UPDATE: Annoyingly the SDK does not include the build targets... But you can download them here. The .NET 4.0 SDK can be downloaded here. Evidentially the .NET 4.0 SDK, or more accurately “Win SDK for Windows 7 and .NET 4”, will not be released until “mid June” (See the last comment from Brian Cost). Need to setup your build server now? No problem, here’s all you have to do (BTW, our developer workstation and build server are both x64 so the paths were the same on both. If you’re going from x86 to x64 or vice versa you’ll have to translate the paths):

  1. Export the following registry key from a box with VS2010 installed and import the keys on the build server (These registry keys contain the paths so as noted above you may need to adjust them if your going from x86 to x64 or vice versa): HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Microsoft SDKs\Windows\v7.0A
  2. Copy the following path from the same machine above to the corresponding path on the build server: %ProgramFiles%\Microsoft SDKs\Windows\v7.0A
    3.
  3. If you need to compile web application projects you need to copy the build target from your dev machine or download them here and copy them to the build server here: %ProgramFiles%\MSBuild\Microsoft\

That should be all there is too it.
